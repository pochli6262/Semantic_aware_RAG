## ğŸ§  Project Overview

This project introduces a **Semantic-Aware Retrieval-Augmented Generation (RAG)** framework for **automated single-line error repair** in competitive programming.
Unlike traditional Automated Program Repair (APR), this method combines **problem descriptions** and **buggy code embeddings** to guide retrieval and enhance repair accuracy.

---

## ğŸ—ï¸ System Architecture

The framework operates in five main stages:

1. **Bug Summary Generation** â€“ Generated by an oracle or LLM.
2. **Retrieval** â€“ Problem descriptions and buggy code are embedded into a FAISS index.
3. **RAG Augmentation** â€“ Retrieves the top-k most relevant repair examples.
4. **Repair Generation** â€“ LLM generates fixes based on code and retrieved examples.
5. **Validation** â€“ Candidate fixes are tested against problem-specific cases.
